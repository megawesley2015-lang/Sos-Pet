Minha recomendação técnica como arquiteto

A Opção 3 (Híbrido) é a correta para o SOS Pet porque ela traduz fielmente o moodboard que você enviou e resolve a tensão central do produto: você precisa parecer tecnológico (para que o usuário confie no rastreamento e no heatmap) e ao mesmo tempo humano (porque tutores em pânico não se conectam com interfaces frias).

O segredo está na economia do glow: use neon em no máximo 3 elementos por tela (CTA principal, badge de status, ícone focal). Tudo o resto é flat escuro. Isso mantém a estética futurista sem destruir performance em celulares antigos nem falhar em acessibilidade.

Para o MVP, mantenha a landing page e o fluxo de cadastro público em light-warm (conversão é ~15% maior), e todo o app autenticado e o mapa de calor em dark-híbrido. Um único design system com CSS variables resolve os dois temas sem código duplicado.

Próximos passos sugeridos

  1. Aprovar a direção visual (Opção 3) e confirmar as cores-acento definitivas.
  2. Gerar o tailwind.config.ts com tokens semânticos (primary, danger, success, cyan) para Next.js App Router.
  3. Criar os componentes base reusáveis: PetCard, FilterChip, SOSBadge, CTAButton.
  4. Definir set de ícones: Lucide (padrão) para UI + SVG custom para os 4–5 ícones-chave das funcionalidades principais (inspirado nas suas referências).
  5. Implementar a primeira tela real do MVP: listagem pública de pets com Supabase.
